# Multi Pages Routing

[Reference To Navigator API](https://docs.flutter.dev/ui/navigation)

[Material App Routing: Named Routes](https://docs.flutter.dev/cookbook/navigation/named-routes)


## Biolerplate:

Generally, flutter application would consist of different screens and routes. Which is managed as:

```dart
import 'package:flutter/material.dart';

// screens
import 'package:world_time/pages/home.dart';
import 'package:world_time/pages/loading.dart';
import 'package:world_time/pages/choost_location.dart';

void main() {
  runApp(MaterialApp(
    initialRoute: '/home',
    routes: {
      '/': (context) => Loading(),
      '/home': (context) => Home(),
      '/choose_location': (context) => ChooseLocation(),
    },
  ));
}
```

``MaterialApp`` takes two argument: ``initialRoute`` and ``routes``.

- ``initialRoute`` define which route is called first.
- ``routes`` is a map of routes.
    - The key is the route name. In our case it's ``'/'`` (home), ``'/home'`` and ``'/choose_location'``.
    - The value is the function that will be called when the route is called.
    - Note: Functions are defined using ``() => Widget`` syntax.
    - The function takes ``context`` argument which is used to navigate to other routes. for e.g:

    ```dart
    '/home': (context) => Home(),
    ```


## Navigating Between Pages:

Suppose we have a button to choose a location on a home page and we want tp navigate to ``choose_location`` page. So we would write:

```dart
    Navigator.pushNamed(context, '/choose_location');
```

``Navigator.pushNamed`` takes a context as a first argument and the name of the route as a second argument.

```dart
TextButton.icon(
  onPressed: () {
    Navigator.pushNamed(context, '/choose_location');
  },
  icon: Icon(Icons.edit_location),
  label: Text('Edit Location'),
)
```

Note that, when we navigate to the new screen, previous screen is still in the memory, just the thing is that, the current screen is at the top
and the previous screen is just below in the stack. To go back, we can use the back button provided by the navigator, which is just above the appbar. 

```dart
  Navigator.pop(context);
```

## pushReplacementNamed:

Other method is ``Navigator.pushReplacementNamed`` which is used to replace the current route with the new route. 

Note that ``pushReplacementNamed`` is different from pushNamed in the sense that it removes the current route from the stack.

```dart
    Navigator.pushReplacementNamed(context, '/home', arguments: {
      'location': wt.location,
      'flag': wt.flag,
      'time': wt.time,
    });
```
 
 Here, ``argument`` is the data passed to the route. We can access the argument in the ``build`` method of the widget using:

 ```dart
 Map data = ModalRoute.of(context)!.settings.arguments as Map;
 ```

