# Widget LifeCycle


[Reference To Stateful Widget Lifecycel](https://www.darttutorial.org/flutter-tutorial/flutter-stateful-widget-lifecycle/)

We have two types of widget in Flutter.

1. Stateless Widget
2. Stateful Widget


**Stateless widgets** are widgets that do not have any state. They are immutable and their properties cannot be changed.

**Stateful widgets** are widgets that have state. They are mutable and their properties can be changed.



## Lifecycle Of Stateful Widget

**1. initState:**
- Called only once when the widget is created.

**2. build:**
- Builds the widget.
- Called every time the state changes, or whenever we use ``setState``

**3. dispose:**
- Called when the widget / state is removed from the widget tree.




```dart

class ChooseLocation extends StatefulWidget {
  const ChooseLocation({super.key});

  @override
  State<ChooseLocation> createState() => _ChooseLocationState();
}

class _ChooseLocationState extends State<ChooseLocation> {

  // initState() is called only once when the widget is created.
  @override
  void initState() {
    super.initState();
    print('Init state called');
  }

  // dispose() is called when the widget / state is removed from the widget tree.
  @override
  void dispose() {
    super.dispose();
    print('Dispose called');
  }

// Called when building widget, or everytime when state changes, setState is called
  @override
  Widget build(BuildContext context) {
    print('Build called');
    return Scaffold(
      ...
      ...
    );
  }
}
```
